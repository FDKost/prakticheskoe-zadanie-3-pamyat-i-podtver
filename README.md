# Практическое задание №3: Память и подтверждение действий

## Описание

В этом проекте реализован агент на базе LangGraph, который:
- Сохраняет историю диалога в памяти (`MemorySaver`).
- При вызове инструмента (например, калькулятора) делает паузу и запрашивает подтверждение пользователя.
- Позволяет пользователю отменить вызов инструмента.
- Сохраняет состояние между перезапусками.

## Как это работает

1. **MemorySaver**  
   При каждом сообщении пользователь/ассистент добавляется в память.  
   После каждой итерации состояние сохраняется с помощью `memory.save()`.

2. **Инструмент**  
   В примере реализован калькулятор, который складывает два числа.  
   Вызов инструмента происходит через узел `tools` в графе.

3. **Interrupt Before**  
   При переходе к узлу `tools` агент генерирует специальный блок `__interrupt__`.  
   В цикле обработки потоков мы ловим этот блок и спрашиваем пользователя, разрешить ли вызов.

4. **Подтверждение**  
   Если пользователь отвечает `Y`, поток продолжается и инструмент вызывается.  
   Если `N`, вызов отменяется и агент завершает работу.

5. **Thread ID**  
   Для каждой сессии генерируется уникальный `thread_id`, который используется для хранения истории в памяти.

## Перезапуск

Поскольку используется `MemorySaver`, после перезапуска агент продолжит диалог с того места, где остановился.
